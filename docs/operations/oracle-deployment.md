# Oracle product deployment and rollback

Status: production-composed CDK contract. This repository does not deploy as part
of validation or review.

## Runtime shape

`OracleFoundationStack` targets AWS account `417242953053`, region `us-east-2`:

| Surface | Product resource | Idle behavior |
|---|---|---|
| Evaluator web | private, versioned, object-locked S3 origin behind CloudFront OAC | storage and requests only |
| Public release artifacts | separate private, versioned, object-locked S3 origin behind its own CloudFront distribution | storage and requests only |
| Restricted artifacts | isolated private, versioned, object-locked S3 bucket | storage only; no API, MCP, or CloudFront grant |
| Application API | Node.js 22 x86_64 Lambda behind explicit HTTP API routes | scales to zero |
| Named-evidence MCP | Node.js 22 x86_64 Lambda at `POST /mcp` | scales to zero |
| Health monitoring | two EventBridge API destinations on a five-minute schedule | two query-free HTTP GETs per interval |
| Agent inference | exact promoted Bedrock profile only | per request; absent without complete promotion inputs |

The stack creates no always-on database, search cluster, instance, container
service, or data job. API and MCP execute the committed production composition
against one verified immutable public release packaged read-only at
`/var/task/release` in their shared Lambda code asset.

Both functions use Node.js 22, x86_64, 1,024 MiB memory, 2 GiB ephemeral storage,
a 30-second timeout, active X-Ray tracing, and 90-day logs. They do not set
reserved concurrency because the target account must retain Lambda's mandatory
unreserved concurrency floor. HTTP API stage throttles bound ingress at burst 20
and rate 10 requests/second.

## Required immutable release input

Production synthesis has no fixture or unconfigured fallback. Supply both CDK
contexts:

| CDK context | Required value |
|---|---|
| `oraclePublicReleaseDirectory` | caller-selected, already verified public release directory beneath this repository root; absolute or repository-relative |
| `oracleServingConfigRelativePath` | portable relative path from that directory to the serving JSON |

Synthesis rejects a missing path, a non-directory, the repository root, an
outside-repository path, a symlink anywhere in the release, path traversal, or
the CDK test-fixture tree. Validation happens before asset creation and requires:

- the exact serving-config and manifest field contracts;
- manifest contract `1.0.0`, Santa Clara County/California identity, and DuckDB
  runtime `v1.4.5`;
- a canonical manifest SHA-256 and matching release/run/hash/as-of values in the
  serving config;
- exactly the six frozen ranking criteria and capability declarations;
- all seven required public relations with their frozen filenames, grains, and
  schema hashes;
- exact artifact byte size and SHA-256 plus bounded Parquet `PAR1` markers;
- no restricted artifact bytes, symlinks, extra closure files, credential-like
  paths, `.config`, sensitive JSON keys, or owner-bearing public columns.

The same validator result is the sole source of both production assets. Lambda
packaging receives its repository-relative directory, while the public-release
deployment receives the validator-returned exact absolute directory. CDK does
not re-read the raw context, select a parent, or discover files independently.
The deployment extracts that validated nine-file closure at the private
`PublicReleaseBucket` root, so the stable artifact paths are
`/release-manifest.json`, `/serving-config.json`, and
`/public/<relation>.parquet`. It never targets the evaluator web bucket or the
restricted bucket. Promotion uses `prune: false`, retains deployed objects when
the deployment resource is removed, and explicitly invalidates `/*` on
`ArtifactDistribution` while waiting for invalidation completion. The bucket
remains private behind CloudFront OAC; the deployment does not add a public ACL,
public bucket policy, or product-runtime write grant.

The ignored directory generated by
`infra/cdk/test-fixtures/create-public-release.mts` contains tiny empty Parquet
relations only for CDK packaging tests. The production app explicitly rejects
that tree. Never pass it to `cdk deploy`, upload it as product data, or treat its
health result as county evidence.

Example production synthesis, using placeholders only:

```powershell

$env:AWS_PROFILE = '<selected-profile>'

aws sts get-caller-identity

  -c oraclePublicReleaseDirectory='<verified-repository-child-directory>' `

```

Stop if Node is not exactly `v22.18.0`, the identity is not account
`417242953053`, or the configured deployment region is not `us-east-2`.

## Lambda asset and native dependency gate

API and MCP share one output-hashed asset. CDK always runs its bundler in a
Linux `amd64` Docker image pinned to Node `22.18.0`, pnpm `10.33.0`, and esbuild
`0.28.1`. The bundler installs from the committed frozen lockfile, bundles the
production API and MCP compositions as `api.mjs` and `mcp.mjs`, and embeds the
validated release under `release/`.

Only the exact native DuckDB closure is externalized and copied:

- `@duckdb/node-api@1.4.5-r.1`;
- `@duckdb/node-bindings@1.4.5-r.1`; and
- `@duckdb/node-bindings-linux-x64@1.4.5-r.1`.

The bundler verifies every package identity and rejects Windows, Darwin, or
ARM64 native package directories. It never copies host `node_modules`. The
opt-in regression gate mounts the synthesized asset read-only in the official
Lambda Node 22 x86_64 image, imports DuckDB, executes `SELECT 1`, imports each
handler, and invokes the query-free health route. Both services must report the
packaged production release ready.

## Runtime environment and stable cursor secret

The production handlers receive:

```text
ORACLE_RELEASE_ROOT=/var/task/release
ORACLE_SERVING_CONFIG_RELATIVE_PATH=<validated relative path>
ORACLE_CURSOR_HMAC_SECRET_BASE64=<CloudFormation Secrets Manager dynamic reference>
ORACLE_ALLOWED_ORIGINS=https://<web CloudFront domain>  # API only
```

`ORACLE_RELEASE_ROOT` is absolute and the asset is read-only in Lambda. The API
origin is one exact HTTPS value; it is never `*` or a comma-expanded fallback.

Secrets Manager generates one stable JSON field named
`cursorHmacSecretBase64`. Its 44-character punctuation-free value is canonical
unpadded base64 representing 33 bytes. API and MCP use the same CloudFormation
dynamic reference, so the secret is neither present in source/synth output nor
fetched by application code. Consequently, neither product role has
`secretsmanager:GetSecretValue`. Rotation must preserve one coordinated value
through a controlled stack update; independent API/MCP secrets invalidate
cursors.

The packaged release removes runtime S3 dependency from API/MCP. Their roles
receive no public-release or restricted-bucket read/write/delete grants. The
public artifact distribution remains a separate browser-facing origin, while
the restricted bucket has no public execution or CloudFront path.

## Routes, CORS, and SPA boundary

The HTTP API exposes only:

```text
GET  /health
POST /{operation}
POST /trpc/{operation}
POST /mcp
GET  /mcp/health
```

There is no `ANY` route. Browser CORS allows only the exact CloudFront web
origin, `GET`/`POST`/preflight, and the committed bounded headers. CORS is not
authorization; the absence of a restricted route is the data boundary.

The web CloudFront Function rewrites only extensionless SPA deep links to
`/index.html`. Requests at or below `/api`, `/mcp`, `/assets`, `/trpc`, or
`/health`; top-level operation paths beginning `dataset.`, `pipeline.`,
`property.`, `inquiry.`, `artifacts.`, or `agent.`; and any file-like path
containing an extension remain origin requests and can return a real miss. UI
routes such as plural `/inquiries/*` remain valid SPA deep links. There is no
distribution-wide 403/404-to-HTML custom error response, so unknown API, MCP,
and asset paths cannot become HTML `200` responses.

The artifact distribution allows browser `GET`/`HEAD`/`OPTIONS` and bounded
range headers only from the web origin. Both distributions redirect to HTTPS
and apply security headers.

## Effect-free health and alarms

EventBridge API destinations invoke only `GET /health` and `GET /mcp/health`.
The committed handlers report zero release queries, DuckDB data queries,
named-evidence calls, and model calls for these probes. Health never invokes a
product operation or MCP tool.

Each probe has two bounded retries and a one-hour maximum event age. Both share
an encrypted 14-day DLQ. DLQ depth, API/MCP function errors, failed probes, and
HTTP API `5xx` responses alarm through the operational SNS topic and notify on
recovery. Subscribe an approved on-call integration to
`OperationalAlertTopicArn` before deployment; the CDK embeds no notification
credential.

## Optional Bedrock promotion

The baseline synth contains no Bedrock permission or model environment. Supply
all five promotion contexts or none:

| CDK context | Required value |
|---|---|
| `oracleBedrockInferenceProfileArn` | exact promoted inference-profile ARN |
| `oracleBedrockInvocationResourceArns` | comma-separated exact profile and required foundation-model ARNs; no wildcard |
| `oracleBedrockModelId` | exact promoted model/inference-profile ID |
| `oracleBedrockRegion` | `us-east-1` or `us-east-2` |
| `oracleAgentPolicyHash` | exact `sha256:<64 lowercase hex>` semantic-policy hash |

Partial input fails synthesis. Complete input grants only the listed resources,
conditioned on the exact inference-profile ARN. Use the GT-O13 promotion
receipt; candidate qualification alone is not promotion evidence.

## Deployment sequence and rollback

Deployment is a separate authorized action. After all gates pass:

1. verify Node, account, region, and the selected immutable release path;
2. run the validation matrix, production `cdk synth --no-lookups`, and `cdk diff`
   with the same release and optional promotion contexts;
3. inspect the synth for dynamic secret references, both x86_64 handlers, the
   release assets, root-targeted public bucket deployment, explicit
   `ArtifactDistribution` invalidation, explicit routes, and the absence of
   restricted bytes;
4. deploy without printing context containing credentials and record the seven
   stable outputs;
5. externally smoke `WebUrl`, `/health`, `/mcp/health`, one real named API query,
   one MCP tool, a SPA deep-link refresh, API/MCP/assets misses, and public
   artifact byte/hash/range behavior;
6. confirm the release ID, manifest hash, capabilities, and cursor continuity;
   and
7. retain the prior release directory, deployed asset, and web object versions
   until hosted checks pass.

Web deployment uses `prune: false`, retained object versions, seven-day object
lock, and retained buckets. Public/restricted buckets use 30-day object lock.
CloudFormation rollback cannot erase prior evaluator or release bytes. Roll back
behavior by redeploying the last verified infrastructure/web revision with its
exact prior immutable release and secret continuity. Never edit a released
bundle in place or substitute the CDK fixture.

If API/MCP health or a named query fails, restore the prior verified stack/asset,
disable optional inference if necessary, and inspect alarms and the DLQ. A green
query-free health probe proves process and release composition only; it does not
replace real-query hosted smoke.
